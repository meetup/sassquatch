Nav Buttons

Using a list of <a> elements; for toggling/filtering controls.

Default display

  • Date Joined
  • Size
  • Name
  • Role
<ul class="nav-buttons">
        <li><a href="#">Date Joined</a></li>
        <li><a href="#" class="selected">Size</a></li>
        <li><a href="#">Name</a></li>
        <li><a href="#">Role</a></li>
    </ul>

Vertical variant

With nav-buttons--column variant class: html_example <ul class="nav-buttons nav-buttons--column"> <li><a href="#">Date Joined</a></li> <li><a href="#" class="selected">Size</a></li> <li><a href="#">Name</a></li> <li><a href="#">Role</a></li> </ul>

Segmented Control

This is a design pattern that should be used for elements that appear as buttons in the UI and have button-like functionality, appearing grouped together and interrelated.

Element type Use
<a> Hash modification; changing contents of view below
<input radio> Making a mutually-exclusive form choice
<input checkbox> Toggling a particular feature 
<div class='segmented-control'>
        <div class="segmented-control-segment">
            <input type='radio' name='guestsRadio' value='0' id='guestsRadio0' class='segmented-control-input' checked />
            <label for='guestsRadio0' id='guestsRadio0label' class='segmented-control-button'>Just me</label>
        </div>
        <div class="segmented-control-segment">
            <input type='radio' name='guestsRadio' value='1' id='guestsRadio1' class='segmented-control-input' />
            <label for='guestsRadio1' id='guestsRadio1label' class='segmented-control-button'>+1</label>
        </div>
    </div>
